How it works & honest caveats

Each idea is scored by a transparent model: Hit Score = f(demand × trend momentum, virality coefficient, −competition, monetization, feasibility) + discovered synergies, self-calibrated so the average idea ≈ 50. The agent uses policy-gradient RL (REINFORCE): it samples whole ideas, and for choices that beat its running average it raises their probability. It reliably converges on the top ~0.01% of a 46,656-combination space.

Data grounding: checking… The pipeline is wired to consume real numbers — each feature maps to a target keyword, and demand comes from real search volume while competition comes from real keyword difficulty as soon as a keyword_data.js table is supplied (see tools/build_keyword_data.mjs).

Caveat: where a feature isn't grounded, its demand and competition are heuristic estimates — a structured hypothesis engine, not a market oracle. Treat the output as a ranked set of hypotheses to validate, not a guarantee.

Policy-gradient RL over a combinatorial idea space. No frameworks, no web search.
